**Постановка задачи**

Разработать программную систему, моделирующую игрового персонажа и демонстрирующую различие между передачей объекта в функцию по указателю и по значению.

Программная система содержит один класс. Класс должен включать:

- одно закрытое целочисленное свойство значения здоровья персонажа;
- открытые методы установки и получения значения здоровья;
- открытый метод изменения значения здоровья (значение параметра прибавляется к значению здоровья).

**Требования к реализации**
1. В программе должна быть реализована функция, принимающая указатель на объект.
2. В программе должна быть реализована функция, принимающая объект.
3. В обеих функциях должно использоваться одинаковое условие изменния значения здоровья:
    - если значение здоровья меньше 50, здоровье увеличивается на 10;
    - если значение здоровья больше или равно 50, здоровье уменьшается на 5.
4.     Освобождение динамически выделенной памяти является обязательным.
5.     Использование передачи по ссылке запрещено.

**Алгоритм работы программы**
1) Динамически создать объект класса.
2) Ввести начальное значение здоровья.
3) Установить значение здоровья через метод класса.
4) Передать указатель на объект в функцию.
5) Передать объект в функцию.
6) Вывести итоговое значение здоровья.
7) Освободить выделенную память.
8) Завершить работу программы.
9) При решении задачи необходимо руководствоваться методическим пособием и приложением к методическому пособию

**Входные данные**
```powershell
Одно целое число — начальное значение здоровья персонажа.
```
**Выходные данные**
